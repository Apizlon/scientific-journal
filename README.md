# scientific-journal

## Запуск (Docker)
- Требуется: Docker + Docker Compose.
- Данные: лежат на хосте в `./data/` (папка в `.gitignore`) и переживают перезапуск контейнера.

### Старт
```bash
docker compose up --build
```

Откройте:
- `http://localhost:8080/` (вход)

### Тестовые пользователи
- Администратор: `admin / admin`
- Редколлегия: `redact / redact`

### Структура
- `static/` — статичные страницы (`*.html`)
- `assets/` — стили/скрипты/картинки
- `perl/cgi/` — серверные CGI-скрипты (`*.pl`, доступны как `/cgi-bin/...`)
- `perl/lib/` — общие Perl-модули
- `data/` — файлы базы (монтируется в контейнер как `/data`)

### Сессии
Сессии хранятся в таблице `sessions` (SQLite), клиент получает cookie `sj_session` на 7 дней. При каждом запросе к `/cgi-bin/...` сессия проверяется.

### Файлы .pm
Это perl-модули, они хранят общую логику и переиспользуются CGI-скриптами
1; в конце файлы - это требование Perl‑модулей: файл должен вернуть истинное значение при use/require. 1; — стандартная практика, чтобы модуль корректно загружался.

### SQLite
SQLite подключается через DBI (DSN dbi:SQLite:dbname=/data/app.db).

База создаётся и наполняется тестовыми данными при первом обращении к любому CGI‑разделу (`/cgi-bin/...`): вызываются `ensure_schema` и `seed_if_empty` в `Journal::DB`.

Основные методы:
- $dbh->do(...) — создание/изменение схемы и простые запросы без выборки.
- $dbh->selectrow_array(...) — получить одну строку (скаляр/массив).
- $dbh->selectrow_hashref(...) — получить одну строку как хеш.
- $dbh->selectall_arrayref(...) — получить множество строк как массив хешей.
